**Learning:** React 18 `act()` wrappers are essential not only for testing state changes from events, but also when executing components that update state based on internal Promises or intervals (like polling stats in `useRealP2PStats`). When testing React context providers that wrap multiple states (like `P2PContext` managing both settings and high-frequency stats), it's highly beneficial to split the context internally and test the memoization bounds to ensure stable components don't re-render excessively.
**Action:** When testing hooks that integrate with socket events or polling, ensure explicit unmounting is tested to verify cleanup (e.g., `socket.off`). For contexts, test the fallback values used when the context hasn't been initialized fully or when accessed outside the provider (by verifying expected thrown errors).
