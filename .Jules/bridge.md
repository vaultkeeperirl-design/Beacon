Redundant joins should be handled on the backend to prevent duplicate signaling events caused by React component remounting or multiple components sharing a single socket. **Action:** Always validate room state on the backend before emitting join/leave notifications.

## 2025-05-22 - [Account Deletion Data Integrity] **Learning:** When deleting a resource (e.g., a User) that has denormalized counters (e.g., `follower_count`) referencing it elsewhere, simply deleting the primary record and its direct associations (e.g., `Follows`) is insufficient. ## 2025-05-23 - [Input Validation for Array Access] **Learning:** In Node.js/JavaScript, accessing an array with a non-integer index (like a float or null) returns `undefined`. Attempting to access or modify a property on this `undefined` result (e.g., `array[1.5].property++`) throws a `TypeError`, which will crash the process if it occurs in a callback (like a Socket.io event handler) without a global try-catch. Range checks (`index >= 0 && index < length`) are insufficient if the type of `index` is not guaranteed to be an integer. **Action:** Always use `Number.isInteger()` to validate indices received from external clients before using them for array access in critical logic paths.
